Firecracker MicroVMs, dasar-dasar

Oke, kita mulai tutorial Firecracker MicroVM dari nol dengan asumsi kamu sudah punya:

  • hello-vmlinux.bin → kernel ELF uncompressed
  • hello-rootfs.ext4 → root filesystem minimal

Kita jalankan step-by-step sampai microVM boot ke shell.

1. Jalankan Firecracker

Bersihkan socket lama lalu jalankan Firecracker:

rm -f /tmp/firecracker.socket
firecracker --api-sock /tmp/firecracker.socket

Biarkan terminal ini terbuka. Firecracker sekarang “nunggu” konfigurasi lewat API.

2. Set kernel (boot-source)

Di terminal baru:

KERNEL_PATH=$(pwd)/hello-vmlinux.bin

curl --unix-socket /tmp/firecracker.socket -i \
    -X PUT 'http://localhost/boot-source' \
    -H 'Content-Type: application/json' \
    -d "{
        \"kernel_image_path\": \"${KERNEL_PATH}\",
        \"boot_args\": \"console=ttyS0 reboot=k panic=1 pci=off\"
    }"

Kalau berhasil, keluar HTTP/1.1 204.

3. Set root filesystem (drives)

Masih di terminal yang sama:

ROOTFS_PATH=$(pwd)/hello-rootfs.ext4

curl --unix-socket /tmp/firecracker.socket -i \
    -X PUT 'http://localhost/drives/rootfs' \
    -H 'Content-Type: application/json' \
    -d "{
        \"drive_id\": \"rootfs\",
        \"path_on_host\": \"${ROOTFS_PATH}\",
        \"is_root_device\": true,
        \"is_read_only\": false
    }"

4. Set konfigurasi mesin (machine-config)

curl --unix-socket /tmp/firecracker.socket -i \
    -X PUT 'http://localhost/machine-config' \
    -H 'Content-Type: application/json' \
    -d '{
        "vcpu_count": 1,
        "mem_size_mib": 128,
        "smt": false
    }'

5. Start MicroVM

curl --unix-socket /tmp/firecracker.socket -i \
    -X PUT 'http://localhost/actions' \
    -H 'Content-Type: application/json' \
    -d '{
        "action_type": "InstanceStart"
    }'

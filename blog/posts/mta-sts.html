Cara benerin error MTA-STS policy is missing: STSFetchResult.NONE

Error "MTA-STS policy is missing: STSFetchResult.NONE" terjadi karena domain email Anda belum mengonfigurasi kebijakan MTA-STS (Mail Transfer Agent - Strict Transport Security) dengan benar. MTA-STS adalah standar keamanan email yang membantu mencegah serangan Man-in-the-Middle (MITM) dengan memaksa komunikasi terenkripsi antara server email.

Cara Memperbaiki Error MTA-STS policy is missing

Berikut langkah-langkah untuk memperbaiki error ini:

  1. Buat DNS Record untuk MTA-STS

    2. Tambahkan record berikut di DNS domain Anda:

    Tipe TXT untuk MTA-STS

    Host: _mta-sts.yourdomain.com

    Value: v=STSv1; id=20240228;

  2. Buat File Policy MTA-STS

    3. Buat file policy.txt dengan isi berikut:

    version: STSv1
mode: enforce
mx: mail.yourdomain.com
max_age: 86400

    Penjelasan:

    • version: STSv1 → Versi kebijakan.
    • mode: enforce → Mengaktifkan kebijakan secara ketat (bisa diganti testing untuk pengujian).
    • mx: mail.yourdomain.com → Ganti dengan server MX yang digunakan oleh domain Anda.
    • max_age: 86400 → Masa berlaku kebijakan dalam detik (1 hari).
  3. Hosting File Policy di HTTPS

    4. File policy.txt harus dihosting di:

    https://mta-sts.yourdomain.com/.well-known/mta-sts.txt

    Jika Anda memiliki server sendiri, buat direktori .well-known dan simpan file di dalamnya:

    /var/www/html/.well-known/mta-sts.txt

    Pastikan bahwa:

    • File dapat diakses di HTTPS.
    • Tidak ada redirect ke HTTP.
    • Menggunakan sertifikat SSL yang valid.
  4. Periksa Konfigurasi dengan Alat Pengecekan

    5. Setelah selesai, gunakan alat berikut untuk memverifikasi konfigurasi:

    • Google Admin Toolbox Check MX → https://toolbox.googleapps.com/apps/checkmx/
    • MTA-STS Validator → https://aykevl.nl/apps/mta-sts/

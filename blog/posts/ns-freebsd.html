Nameserver sendiri di FreeBSD? Kenapa engga!

Cara konfignya gini

Install bind di FreeBSD

pkg install bind916

Enable bind di /etc/rc.conf

nano /etc/rc.conf

named_enable="YES"

Bikin zona example.com

nano /usr/local/etc/namedb/named.conf

Tambahin

zone "example.com" {
    type master;
    file "/usr/local/etc/namedb/master/example.com.zone";
};

Buat file zona

nano /usr/local/etc/namedb/master/example.com.zone

Kalo ga ada foldernya bikin dulu mkdir -p /usr/local/etc/namedb/master

Isi file /usr/local/etc/namedb/master/example.com.zone

$TTL 86400
@   IN  SOA ns1.example.com. admin.example.com. (
           2025032901  ; Serial
           3600        ; Refresh
           1800        ; Retry
           1209600     ; Expire
           86400       ; Minimum TTL
)
@       IN  NS      ns1.example.com.
@       IN  NS      ns2.example.com.
ns1     IN  A       203.0.113.10  ; Ganti dengan IP publik VPS
ns2     IN  A       203.0.113.11  ; Opsional, bisa VPS lain
www     IN  A       203.0.113.20

Abis itu restart bind, service named restart

Daftarin Nameserver di Registrar

  1. Bikin dulu record DNS dengan tipe NS, nama domain di sesuaikan
  2. Bikin private nameserver dengan nama domain yang sama
  3. Abis itu pointing nameserver bikinan sendiri di bagian setting nameserve

Contoh:

Di domain registrar, tambahin glue record

  • ns1.example.com → 203.0.113.10
  • ns2.example.com → 203.0.113.11

Lalu, ubah nameserver domain ke:

  • ns1.example.com
  • ns2.example.com

Uji Coba Nameserver

Pake perintah ini

dig @203.0.113.10 example.com

Atau cek propagasi pake:

dig NS example.com +trace

Ada beberapa lagi perintah dig buat cek gituan
